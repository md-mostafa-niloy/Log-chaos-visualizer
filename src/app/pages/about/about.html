Supported Log Formats

Pino

Node.js JSON logger with structured logging, request/response tracking, and metadata support.

Winston

Popular Node.js logging library with metadata objects and distributed tracing support.

Loki/Promtail

Grafana's log aggregation format with labels and time-based indexing.

Docker

Docker container logs with stream distinction (stdout/stderr).

Plain Text

Syslog-style text logs with automatic parsing and level detection.

Unknown JSON

Generic JSON format fallback for custom log structures.

Advanced Query Language

The application includes a powerful, structured query language for advanced log filtering and analysis:

Query Examples

  • level=error - Find all error-level logs
  • environment=prod AND level=error - Production errors only
  • contains(message, "timeout") - Find timeout messages
  • statusCode>=500 - Server errors (Pino HTTP fields)
  • (level=error OR level=fatal) AND environment=prod - Critical production issues

Supported Fields

  • Common: level, message, timestamp, environment, kind, hostname
  • Pino: msg, time, pid, name, method, url, statusCode, responseTime
  • Winston: requestId, userId, traceId
  • Loki: job, instance, app, line
  • Docker: log, stream

Query Features

  • Field-specific searches
  • Comparison operators (=, !=, >, >=, <, <=)
  • String functions (contains, startsWith, endsWith, matches)
  • Boolean operators (AND, OR, NOT)
  • Regular expression support
  • Parentheses grouping
  • Real-time validation with error messages
